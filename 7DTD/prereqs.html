Folder Paths

You should remember a few folder paths.
Once you find all the Folder Locations

Pin them to Quick Access.
- Right Click on folder > Pin to Quick Access.
Vanilla Config, Mods, ConfigsDump, & ItemIcons.

Vanilla Folders


Config Folder

Where ever your Steam folder is.
This folder holds all the XML files we are going to edit.

Most Common Path


ItemIcons Folder

Where ever your Steam folder is.
This folder is where the games icons are stored.

Most Common Path


ConfigsDump Folder

This folder is to help troubleshoot any problems.
It has XML files that game save files load.

You check this to see if your code edits loaded properly etc.

Search for the AppData Folder
use WIN+R and Copy & Paste.
Then you look for
WorldName\SaveName\ConfigsDump

Example:
Navezgane\A22Testing\ConfigsDump

Manually Setting Up

First make a new folder in the "Mods" folder.
Name the folder your Mods name.
Example: UserNameChanges

Then in that folder make a folder called "Config".

In the "Config" folder add/make whatever XML files you want to edit.
If you want to Edit or Add an Item add the "items.xml" etc.

Inside an XML file you will have to first
write a code block to wrap around your code changes

If you look at any of the Vanilla XML files you will see that
all the code is wrapped in starting tags.
basically <blocks>,<items> etc.
<configs>
Your Code Inside
</configs>
You can name it anything doesn't have to be "configs".

I name it after the XML file i am editing.
As it's easier to know what file I am editing.

Some people name it after themselves, like their username.
Start and End must match.

ModInfo.xml (REQUIRED)

Then will have to add a "ModInfo.xml" file.
The "ModInfo.xml" file is required for the mod to run.

Setting Up Visual Studio Code

Visual Studio Code has a lot you can do,
as such it can be used right out of the box,
however I would like to show you some things
to optimize your efficiency.

Changing the Theme

Bottom Left Corner > Gear Icon
Themes > Color Themes
A Drop Down menu should pop up.
I will forever hate you if you don't use a Dark Theme.

Recommend Themes

GitHub Theme (GitHub) (Dark/Dimmed)
Material Icon Theme (Philipp Kief)

Making a Workspace

Workspaces are great for organization.
I make them for whatever I am working on.
Different websites or Mods.

Drag and Drop 2 folders into Visual Studio Code.
- the Mods folder and the Vanilla Config Folder

Dialog boxes should pop up saying
Add to Workspace and one that says Trust, click Yes and Trust.

Saving the Workspace

File > Save Workspace As...
I suggest saving this in your Documents Folder and
then a folder called VSC or something.

Turn of Auto Save

File > Auto Save
This will save your work every so often automatically.

Work Flow - Split View

The way I work is i Split View the files i work on.
With 7 Days mods
I have the Mod files on the left side
and then the Vanilla files on the Right.
It makes it easier for me to find and reference files.

Basically Drag files onto the far right side to split.

Extensions

Here is my recommended Extensions.
These make my life so much easier!

#region folding for VS Code
This extension is optional as #region is already there by default.
It enhances the default code
folding abilities of Visual Studio Code editor.

Regions of code that you'd like to be folded
can be wrapped with #region / #endregion comments.

Auto Close Tag (Jun Han)
Automatically add HTML/XML close tag.

Auto Rename Tag (Jun Han)
Automatically rename paired HTML/XML tag.

Code Spell Checker(Street Side Software)
A basic spell checker that works well with code and documents.

Context Menu Toggle Comments (henryclayton)
This extension will let you toggle block & line comments
from the explorer context menu, like Notepad++.

Dumb copy-paste (Shepelev Stas)
Dumb simple implementation of ctrl+shift+v from Sublime:
Just select a block of code, hit ctrl+c/ctrl+x,
move curret to where you want it pasted, ctrl+shift+v,
BAM! it is pasted keeping copied/cut
block of code indentation shape.

Fold Plus (dakara)
Toggles between indentation and language folding.
This command is provided to easily toggle between each mode.

TabOut (Albert Romkes)
Tab out of quotes, brackets, etc.

Text Power Tools (Dániel Tar)
Text Power Tools is an all-in-one text manipulation extension for VS Code. Inspired by TextFX for Notepad++ and Filter Lines
and Text Pastry for Sublime Text.

XML (Red Hat)
This VS Code extension provides support
for creating and editing XML documents,
based on the LemMinX XML Language Server.

XML Tools (Josh Johnson)
XML Formatting, XML Tree View, XPath Evaluation, XQuery Linting,
XQuery Execution, XQuery Code Completion,
