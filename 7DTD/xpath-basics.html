XPath Overview

XPath uses path expressions to select
nodes or node-sets in an XML document.

These path expressions look very much like
the path expressions you use with a PC file systems

Basically think of it like Folder Tree-like structure.

Functions

7 Days has XML code functions
you can use to change the Vanilla values.

It doesn't actually touch and change the game files.
The game just reads the files you made and changes it accordingly.

This is what one of a 7 Days to Die XML Function looks like.

Check out the Basic Syntax tab to learn more.

anything between
double quotes "..." or single quotes '...' are String Text

XPath Conditionals

In 99% of cases you will want to work on a specific node
or at least a carefully chosen subset.

That is where conditionals come in.
Since everything descriptive in 7D2D XML is put into an attribute,
you will nearly always be using conditionals on attributes,
like looking for a particular item name or block name.

XPath Basics Notes

Make sure when you are making edits
you are in the right XML file.
Blocks.xml - /blocks/block
Items.xml - /items/item
Recipe.xml - /recipes/recipe
Etc.

Remember there is easy ways to test if your changes works/applied.

1. In-Game Console

When loading the game press F1 to open the console.
- It will show Warnings of things not applying in Yellow text
- As well as Errors in Red Text

2. ConfigsDump Folder

This folder is to help troubleshoot any problems.
It has XML files that game save files load.

You check this to see if your code edits loaded properly etc.

Search for the AppData Folder
use WIN+R and Copy & Paste.
Then you look for
WorldName\SaveName\ConfigsDump
Example:
Navezgane\A22Testing\ConfigsDump

<Append></Append>

Is a TAG that places stuff at the very bottom of a list of things.
Which depending on what you're trying to do the position of nodes matters.

Example #1

<!-- Add a chance to get Water or Yucca Juice from a Cactus -->
<append xpath="/blocks/block[@name='treeCactus01']">
<drop event="Destroy" name="drinkJarYuccaJuice" count="1,2" prob="0.10"/>
<drop event="Destroy" name="drinkJarBoiledWater" count="1,2" prob="0.05"/>
</append>

Code Explanation

<!-- Add a chance to get Water or Yucca Juice from a Cactus -->
This is a comment. It will not be read by the game.
Basically used for Notes or a TODO list.
SAFE to delete.
Good practice to leave these for yourself.
<append xpath="/blocks/block[@name='treeCactus01']">
<Append> TAG is a function in XML.
It basically adds to the end of list of things.
xpath="/blocks/block[@name='treeCactus01']"
xpath="" is another function
that looks for a specific thing in a specific directory.
It finds a Path in XML.
"/blocks/block[@name='treeCactus01']"
Is the Pathway we are targeting.
In the Vanilla blocks.xml the nodes are wrapped inside
top level node called <blocks>.

So with use the forward slash /
to tell the game to look into that node.

Then after the node name we do another forward slash /
tells the game to look at the next thing
which is a block with the name attribute treeCactus01.
<drop event="Destroy" name="drinkJarYuccaJuice" count="1,2" prob="0.2"/>
With this line we are adding a Node called Drop
Which you can probably guess what it does.

On the block Destroy
it adds the item Yucca Juice to the loot list
and how many it can drop 1 to 2.
and the probability of it dropping which .2 is 20%.
</append>
Is the closing end tag.
Not all Element Nodes have an ending tag.
So you just add a forward slash / at the end.
As seen in the drop tag.

<remove/>

Will remove whole nodes or block of codes.

Example #1

Say you want to remove an ingredient from a recipe.
Let's make Steroids not use Oil to craft.
<remove xpath="/recipes/recipe[@name='drugSteroids']/ingredient[@name='resourceOil']"/>

Results

Now the Steroids doesn't need Oil.

Remove Notes

Remove Notes

<removeattribute/>

Removes a node's attribute.

Example #1

Let's make Ladder Steel not require a Workbench to craft.
<removeattribute xpath="/recipes/recipe[@name='ladderSteel']/@craft_area"/>
This isn't necessarily needed but just makes sense.
<removeattribute xpath="/recipes/recipe[@name='ladderSteel']/@tags"/>

Results

Now the Ladder Steel doesn't need a Workbench to craft it.

<set></set>

Sets/Changes already existing Nodes.

Example #1

Say you want to change the time it takes to craft something.
Let's make the Land Claim Block take less time to craft.
<set xpath="/recipes/recipe[@name='keystoneBlock']/@craft_time">1</set>

Results

Now the Land Claim Block takes 1 second instead of 5.

Example #2

Say you want to change the amount of a recipe ingredient.
Let's make the Boiled Meat use less meat.
<set xpath="/recipes/recipe[@name='foodBoiledMeat']/ingredient[@name='foodRawMeat']/@count">1</set>

Results

Now the Boiled Meat requires 1 meat instead of 5.

<setattribute></setattribute>

Can set/change existing attributes but
should be used when adding a new Attribute.

Example #1

Say you want to add an attribute, like make a recipe require a Station.
Let's make the Barbed Wire Fence require a Workbench to craft.
<setattribute xpath="/recipes/recipe[@name='barbedFence']" name="craft_area">workbench</setattribute>

Results

Now the Barbed Wire Fence requires a Workbench to craft.
